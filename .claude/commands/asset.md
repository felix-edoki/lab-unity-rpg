# /asset — Code Quality Audit

Audit the C# code under `$1` (default: `Assets/Scripts/`) against these rules, then write a Markdown review document the user can open and read.

## Rules to verify

1. **Function length** — no function (method, constructor, local function, lambda with block body) longer than **40 lines** of code (count from opening `{` line through closing `}` line inclusive).
2. **No nested ternary operators** — a `?:` expression must not contain another `?:` in either its true-branch or false-branch (directly or within parenthesized sub-expressions).
3. **Parameter count** — no function with more than **10 parameters**.

## How to run the audit

- Scope the scan to `.cs` files under the given path (default `Assets/Scripts/`). Exclude `Library/`, `Temp/`, `Build/`, `obj/`, `Packages/`, and auto-generated files (e.g. containing `// <auto-generated>`).
- Use `Grep` / `Read` to gather evidence. For each violation record: file path, line number, function name (if applicable), and a short excerpt.
- Be precise — do not flag things that are not actual violations. When in doubt about a function length, `Read` the relevant range to confirm.

## Output

Write the report to `ASSET_REVIEW.md` at the project root (overwrite if it already exists). Use this structure:

```
# Asset Code Review — <YYYY-MM-DD>

Scope: `<path scanned>`
Files scanned: <N>

## Summary

- <emoji> Function length (≤ 40 lines): <count> violation(s)
- <emoji> No nested ternaries: <count> violation(s)
- <emoji> Parameter count (≤ 10): <count> violation(s)

## Checklist

- ✅ / ❌ Rule 1 — Functions ≤ 40 lines
- ✅ / ❌ Rule 2 — No nested ternary operators
- ✅ / ❌ Rule 3 — Functions ≤ 10 parameters

## Details

### 📏 Functions longer than 40 lines
<list or "✅ None">

### 🔀 Nested ternary operators
<list or "✅ None">

### 📦 Functions with more than 10 parameters
<list or "✅ None">
```

Rules for emojis in the Summary:
- ✅ when the count is 0
- ⚠️ when 1–5
- ❌ when > 5

Each violation in **Details** should follow this format:

```
- `relative/path/File.cs:LINE` — `FunctionName` — <metric, e.g. "52 lines", "12 parameters", "nested ?: inside return">
```

After writing the file, tell the user the path to `ASSET_REVIEW.md` and print the one-line Summary block in your response. Do not dump the full report into the chat.
